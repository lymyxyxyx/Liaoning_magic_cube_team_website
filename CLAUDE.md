## Architecture

This is a **Next.js 14 App Router** site for the Liaoning Rubik's Cube team. Data comes from three distinct sources that must not be confused:

### 1. Static TypeScript data (source of truth for team members and competitions)

- `lib/data.ts` ‚Äî Core people (`corePeople`), competitions, achievements, and relations. The `people` export merges `corePeople` with `weeklyPlayerPeople`.
- `lib/commercial-teams.ts` ‚Äî Brand-sponsored team members (GAN Gurus, Speed ACE, etc.) and team groupings. Members here also appear as `Person` objects but are managed separately.
- `lib/weekly-players.ts` ‚Äî Auto-generated `Person` stubs for weekly meet participants; do not hand-edit individual bios here.
- `lib/local-profiles.ts` ‚Äî Static seed of WCA ID ‚Üí province/city mappings (used as fallback before PostgreSQL is set up).

### 2. PostgreSQL (runtime data, requires `DATABASE_URL`)

- Weekly meets, events, and results (`lib/weekly-db.ts`)
- WCA sync tables (`wca_persons`, `wca_results`, `wca_ranks_single`, `wca_ranks_average`, etc.) populated by `scripts/update_wca_data.mjs`
- Local province/city profiles (writable via admin API)
- Account book records

### 3. SQLite file (generated locally, never committed)

- `data/wca_rankings.sqlite` ‚Äî Powers the `/rankings` page API. Generated by `scripts/build_wca_china_333_rankings.py` from a WCA official SQL export file. The Python script expects the export at `../WCA_export_v2_*.sql/WCA_export.sql` relative to the project root.

### Two separate ranking pages

- `/rankings` ‚Äî Queries `data/wca_rankings.sqlite` via a Next.js API route. Shows official WCA rankings filtered by country/event/type.
- `/liaoning-rankings` ‚Äî Queries PostgreSQL WCA sync tables joined with local province mappings. Shows Liaoning-specific rankings.

### Auth / middleware

`middleware.ts` guards `/admin/*`, `/api/account-books`, `/api/local-profiles` (non-GET), and `/api/admin/*` using a session cookie (`liaoning_admin_session`). The admin area redirects `www.lncubing.com/admin` to the canonical domain.
