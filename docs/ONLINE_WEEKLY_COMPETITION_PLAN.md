# 辽宁线上周赛规划

## 目标

将目前偏后台维护的周赛，升级为所有辽宁魔方爱好者都能参与的固定线上活动：选手每周自行报名、在规定时间内完成成绩提交、查看分组和总榜排名，通过连续参与、个人进步和赛季积分促进交流。

第一阶段不追求替代 WCA 正式赛事，而是建立一个透明、轻量、可持续的社区周赛。

## 核心原则

- **参与简单**：手机也能完成报名、录入和查看排名。
- **规则清楚**：每周项目、赛制、截止时间和打乱规则，必须在页面固定展示。
- **成绩可信**：成绩先进入待审核状态；公开榜只统计有效成绩。
- **鼓励参与**：除了绝对排名，也展示个人 PB、连续参赛和年龄组排名。
- **可逐步运营**：先由管理员审核，参与人数稳定后再增加志愿裁判和自动化校验。

## 用户与权限

| 角色 | 主要能力 |
| --- | --- |
| 访客 | 查看本周规则、公开榜、历史周赛和选手档案。 |
| 选手 | 直接搜索自己的 WCA ID 或姓名，建立/选择选手档案并提交成绩。 |
| 审核员 | 审核报名信息和成绩，标记有效、驳回或要求补充。 |
| 管理员 | 创建周赛、配置项目与规则、管理审核员、发布公告、处理申诉。 |

首期不要求选手注册或登录，也不复用管理员密码。选手通过姓名或 WCA ID 搜索并选择自己后即可录入成绩；管理员账号仅用于配置周赛、代录、纠错和处理争议。后续若出现捣乱或需要个人通知，再评估引入选手登录系统。

## 每周流程

```text
周一：发布本周周赛与报名
  ↓
周二至周日：选手报名、完成打乱、提交成绩
  ↓
周日截止：系统关闭提交，生成待审核列表
  ↓
周一至周二：审核员处理成绩与申诉
  ↓
发布正式周榜、年龄组榜、积分榜和个人 PB
```

首期预设二阶、三阶、金字塔、斜转、枫叶、镜面与个人全能。每周由管理员决定实际开放项目；个人全能是按“枫叶 → 金字塔 → 二阶 → 三阶 → 镜面”固定顺序连续完成五个魔方的独立计时项目，记录总用时。

## 双通道成绩录入方案

同一套周赛、同一张榜单，但保留两种录入方式：选手自助提交与管理员代录。两者的区别只在于提交人、来源和默认审核状态，最终都落到统一的成绩记录中。

| 场景 | 提交人 | 默认状态 | 适用情况 |
| --- | --- | --- | --- |
| 选手自助录入 | 选手自行选择姓名/WCA ID | 默认有效 | 线上自行参加周赛。 |
| 管理员代录 | 管理员/审核员 | 有效或待审核 | 线下周赛、家长转交成绩、历史补录。 |
| 管理员修正 | 管理员/审核员 | 重新审核 | 处理录入错误、异常成绩复核或申诉。 |

### 选手自助录入流程

1. 选手进入周赛页后，直接通过姓名或 WCA ID 模糊搜索自己，不需要注册或登录。
2. 选择 WCA 选手后自动带入姓名、WCA ID、性别和已有参赛资料；后续参赛可直接按姓名搜索。
3. 如果没有 WCA ID 或找不到选手，补充姓名、性别、年龄组、辽宁城市等资料，建立本地选手档案。
4. 选择已开放项目，查看赛制、截止时间和打乱规则后输入成绩；系统自动计算并校验成绩。
5. 勾选诚信声明后提交；不要求上传录像，成绩默认直接计入榜单。
6. 同一选手、同一周赛、同一项目只保留最新有效提交；系统保留修改记录，方便管理员处理误填或争议。

### 管理员代录流程

现有后台录入页保留并升级为“管理员工作台”：

- 可按姓名、WCA ID 或本地选手库搜索并选择选手。
- 可选择“代录来源”：线下现场、选手提供、历史补录、审核修正。
- 可附管理员备注或原始成绩单。
- 可直接标记为有效；如果来源不是现场，应默认待审核。
- 每次新建、修改、删除都记录操作者、时间、修改前后的成绩与理由。

管理员代录和选手自助录入使用同一个 `weekly_submissions` 表，并通过 `submitted_by_type`、`source`、`review_status` 区分，避免产生两套榜单和不同的成绩计算逻辑。选手自助提交默认有效；管理员可将明显错误、重复或有争议的成绩标记为待复核或撤销。

## 管理员可编辑的周赛配置

管理员每周创建一场周赛，并不再把项目和赛制写死在页面代码中。

### 周赛基础信息

- 标题、期数、开始/截止时间、是否公开、公告内容。
- 适用范围（辽宁主榜、外省体验榜）、参赛资格与审核规则。
- 是否允许自助报名、是否允许自助修改、审核截止时间。

### 项目配置

每个周赛可添加多个项目；每个项目独立设置：

- WCA 项目 ID 与展示名称；首期预设二阶、三阶、金字塔、斜转、枫叶、镜面与个人全能。
- 赛制：Average of 5（Ao5）或三次取最快（Best of 3）。
- 录入次数、是否允许 DNF、成绩单位和排序规则。
- 开放/关闭时间、是否计入积分。
- 打乱信息：文本、链接、打乱编号或“本周自由打乱”。

建议首期允许管理员从预设项目和预设赛制中勾选，而不是完全自由填写公式；这样可防止成绩解析和排名逻辑出现无法处理的组合。

### 推荐后台页面

```text
/admin/weekly
  ├── 周赛列表：创建、复制上周、发布、归档
  ├── 周赛编辑：时间、公告、报名与审核规则
  ├── 项目与赛制：添加项目、选择赛制、配置个人全能和打乱
  ├── 成绩工作台：自助提交队列 + 管理员代录
  ├── 审核队列：有效 / 需补充 / 无效 / 申诉
  └── 发布中心：预览榜单、锁定成绩、发布周报
```

## 在线计时器：建议与边界

**建议首期不把站内计时器作为强制入口，也不把它作为成绩可信的唯一依据。**

原因是线上周赛最难的不是“开始/停止计时”，而是打乱、硬件计时器兼容、误触、成绩防篡改与审核。直接要求所有人使用站内计时器，会增加移动端兼容和参与门槛。

### Phase 1：兼容外部计时器（推荐）

- 选手可用 Stackmat、csTimer、CubeDesk 或手机计时器完成计时。
- 站内只负责录入成绩、校验赛制和展示榜单。
- 以诚信声明、异常成绩复核和管理员裁量维持公平；不要求录像。

### Phase 2：提供站内练习计时器（可选）

将站内计时器定位为练习工具，而非唯一比赛证据：

- 键盘空格、触屏长按/松开、倒计时提示、键盘防误触。
- 可输入或生成打乱，自动保存本地练习记录。
- 支持手动录入 Stackmat 时间；如后续需要，可研究 Web Serial/WebHID 连接硬件计时器，但不作为首期承诺。
- 计时器提交到周赛时，仍要求选手确认成绩与诚信声明。

### Phase 3：比赛模式计时器（仅在运营成熟后）

- 从后台下发每轮固定打乱与唯一编号。
- 计时记录绑定周赛、项目、打乱编号和浏览器时间。
- 异常记录进入人工复核。

即使做到第三阶段，也应允许管理员代录线下/硬件计时器成绩。站内计时器是便利功能，不应让没有合适设备或网络条件的选手失去参赛资格。

## 个人全能规则

个人全能是独立计时项目，不是各项目名次或积分的汇总榜。一次尝试中，选手按“枫叶 → 金字塔 → 二阶 → 三阶 → 镜面”的固定顺序完成五个魔方，记录总用时并按该项目赛制排名。

- 管理员配置五个魔方的打乱和摆放规则；完成顺序固定为“枫叶 → 金字塔 → 二阶 → 三阶 → 镜面”。
- 录入页将五个魔方视为一条完整尝试，提交总成绩；后续可选做分段时间记录，但排名只按总用时。
- 任一魔方 DNF 时，整次个人全能尝试记为 DNF。
- 个人全能与其他项目一样，由管理员选择 Ao5 或三次取最快。

## 选手端功能

### 1. 本周周赛页

- 周赛标题、时间范围、项目、赛制、打乱来源、有效成绩定义。
- 报名按钮和报名状态。
- 截止倒计时；截止后只显示成绩和审核状态。
- 规则摘要：计时器说明、DNF/DNS、诚信声明与申诉时限。

### 2. 成绩提交

- 选择已报名项目，录入每次成绩，由系统自动计算单次、平均和 DNF。
- 勾选诚信声明：成绩为本人按规则完成，并同意公开姓名、成绩和排名。
- 提交后可在截止前修改，保留修改记录。

### 3. 我的周赛

- 当前报名与审核状态：草稿、待审核、需补充、有效、无效、逾期。
- 历史成绩、最好单次/平均、参赛次数、连续参赛周数、积分。
- 收到审核意见时可补充说明或重新提交。

## 榜单与激励

每周至少提供四类可切换榜单：

- **总榜**：按 WCA 对应赛制排序的有效成绩。
- **年龄组榜**：沿用现有少儿/青少年/成人分组。
- **新手榜**：首次参赛或指定成绩阈值内选手，降低初学者挫败感。
- **进步榜**：相对个人历史 PB 的提升幅度或百分比。

赛季可按自然季度累计积分：参与即可得基础分，名次获得额外分，连续参加给予小幅加成。积分需要设置上限，避免单纯“参加次数”压过竞技表现。

## 成绩可信与审核策略

### 首期（人工审核）

- 所有提交默认“待审核”，公开榜只计“有效”成绩。
- 异常快成绩、重复提交或争议成绩进入人工复核；管理员可要求补充说明。
- 审核结论必须带原因：有效、需补充、无效。

### 后续增强

- 每周使用固定随机打乱，页面展示打乱编号和生成时间。
- 自动检测录入格式、异常成绩和短时间多次修改。
- 引入可信选手免审或抽检机制，减少审核压力。
- 建立志愿裁判队列与双人复核，争议成绩由管理员终审。

## 数据模型建议

现有 `weekly-competitions` 和成绩录入能力可继续复用，并补充以下概念：

| 实体 | 关键字段 |
| --- | --- |
| weekly_participants | 用户、选手库条目、报名时间、同意声明、状态。 |
| weekly_submissions | 周赛、项目、五次成绩、自动计算结果、提交/修改时间、审核状态。 |
| weekly_evidence | 成绩提交、备注、审核意见、审核人、审核时间。 |
| weekly_seasons | 名称、起止时间、积分规则、状态。 |
| weekly_points | 赛季、选手、周赛、积分来源、分值。 |

建议新增或扩展字段：

- `weekly_competition_events`：把项目与赛制从周赛主表拆出，可一场周赛多个项目。
- `weekly_submissions.submitted_by_type`：`player` / `admin`。
- `weekly_submissions.source`：`online_self` / `offline_event` / `admin_entry` / `history_import` / `correction`。
- `weekly_submissions.review_status`：`draft` / `pending` / `approved` / `needs_evidence` / `rejected` / `locked`。
- `weekly_submission_revisions`：保留成绩、证据和审核结论的变更历史。

同时补充选手资料能力：WCA ID 可为空；本地选手档案保存姓名、性别、年龄组、所属省份与辽宁城市。姓名/WCA ID 搜索优先匹配已有 WCA 数据和本站本地档案，避免重复建档。

成绩提交与最终榜单建议分离：提交记录保留完整历史，榜单只读取最新且审核有效的记录，便于追溯和处理申诉。

## 分期实施

### Phase 0：规则与运营准备（约 1 周）

- 确定每周固定时间、开放项目、个人全能计分、打乱规则、审核标准和申诉流程。
- 准备选手诚信声明、隐私说明、报名资料说明和 FAQ。
- 招募至少 2 名审核志愿者，避免单人运营中断。

### Phase 1：可公开参与的 MVP（约 2–3 周）

- 公开本周周赛页、姓名/WCA ID 模糊搜索报名和选手身份绑定。
- 允许选手在线提交各开放项目的成绩。
- 增加待审核/有效/需补充状态和管理员审核台。
- 发布有效成绩周榜、年龄组榜和个人历史。

**上线标准**：选手能通过 WCA ID 或姓名完成报名；无 WCA ID 新手能建立本地档案；管理员能代录、审核、驳回和发布；截止后不能篡改成绩。

### Phase 2：持续参与与可视化（约 2 周）

- 我的周赛中心、个人 PB、连续参与和进步榜。
- 周赛归档页、赛季积分榜、周报海报/分享图。
- 审核通知和申诉入口。

### Phase 3：规模化与可信增强（按参与量推进）

- 自动异常检测、抽检、审核员分配与操作日志。
- 固定打乱/随机打乱服务及结果导出。

## 首期需要确认的运营决策

1. 个人全能采用“名次之和越低越好”还是“积分越高越好”？
2. 各项目首期赛制是否统一 Ao5，还是按项目分别设置？
3. 无 WCA ID 新手首次报名时，年龄信息采用出生日期还是直接选择年龄组？
4. 奖励形式：纯线上荣誉、季度证书，还是结合线下活动/小礼品？（待定）

## 推荐的首期默认方案

- 辽宁选手计入主榜；外省选手可参加体验榜并单独标注。
- 首期预设二阶、三阶、金字塔、斜转、枫叶、镜面与个人全能，每周由管理员决定开放项目。
- 不要求录像，以诚信声明、异常成绩复核和管理员审核维持公平。
- WCA ID 优先自动识别；无 WCA ID 新手首次补充资料，后续可按姓名模糊搜索。
- 奖励形式暂定；前四周先发布周榜、个人进步和个人全能，稳定后再确定赛季积分与奖励。

## 已确认的报名与参赛规则

### 参赛范围

- 辽宁选手进入辽宁主榜、个人全能和后续积分榜。
- 外省选手允许报名和提交成绩，显示所属省份，进入单独的体验榜，不影响辽宁选手名次。

### 选手检索与建档

报名页的第一步是一个统一搜索框，可输入姓名或 WCA ID：

1. 先搜索 WCA 数据和本站已经参加过周赛的本地选手库。
2. 选中 WCA 选手后，自动填写姓名、WCA ID、性别等已知信息；选手只需补充所属省份，辽宁选手还需补充城市和年龄组。
3. 选中已有本地选手后，直接带入其已确认资料，方便后续每周快速报名。
4. 搜索不到且没有 WCA ID 时，允许创建本地新手档案；首次填写姓名、性别、年龄组、所属省份、辽宁城市（如适用）和必要联系方式。
5. 本地新手未来获得 WCA ID 后，可由管理员绑定并合并档案，历史周赛成绩不丢失。

公开榜仅显示姓名、所属省份/城市、年龄组和成绩；联系方式、出生日期等资料只供后台审核使用。

### 不要求录像

选手提交成绩时只需勾选诚信声明，不上传录像。出现明显异常、争议或投诉时，由管理员联系选手补充说明，并保留对成绩标记“待复核”或取消有效性的权利。
